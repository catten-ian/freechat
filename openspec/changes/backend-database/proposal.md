# Proposal: 后端数据库存储 + WebSocket 实时传输

## 为什么做（Why）

### 当前问题
1. **数据丢失风险**：前端使用 localStorage，刷新页面或清除缓存会丢失数据
2. **网络不稳定**：前端直接调用 API，网络不好时请求失败，消息丢失
3. **无法恢复**：意外关闭浏览器后，未发送的消息无法恢复
4. **跨设备同步**：不同设备无法同步对话历史

### 用户痛点
- 正在输入长消息时网络断开，消息丢失
- 浏览器崩溃后，所有对话历史消失
- 换设备后，之前的对话无法查看

## 做什么（What）

### 核心功能
1. **SQLite 数据库**：后端持久化存储对话数据
2. **WebSocket 实时传输**：后端接收请求，通过 WebSocket 推送到前端
3. **离线队列**：网络断开时，消息暂存本地，恢复后自动发送
4. **自动重连**：WebSocket 断开后自动重连，恢复会话

### 技术方案
- **数据库**：SQLite（轻量级，无需额外服务）
- **实时通信**：WebSocket（已有基础，扩展功能）
- **离线处理**：前端 IndexedDB + Service Worker
- **重连机制**：指数退避重连策略

## 影响范围

### 后端改动
- 新增 SQLite 数据库表设计
- 扩展 WebSocket 消息类型
- 新增 REST API（对话 CRUD）

### 前端改动
- 移除 localStorage，改用 WebSocket 同步
- 新增离线队列管理
- 新增网络状态监控

### 兼容性
- 保持现有 API 兼容
- 渐进式迁移，不影响现有功能

## 成功标准

1. ✅ 刷新页面后对话历史保留
2. ✅ 网络断开后消息不丢失
3. ✅ WebSocket 断开后自动重连
4. ✅ 跨设备登录可同步对话
5. ✅ 性能：1000+ 对话加载 < 500ms
